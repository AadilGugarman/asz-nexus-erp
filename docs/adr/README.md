# ADR Index

This folder stores Architecture Decision Records for ASZ Nexus ERP.

## How to use

- Create one ADR per significant architectural decision.
- Keep ADRs immutable after acceptance.
- If a decision changes, add a new ADR that supersedes the old one.

## Status definitions

- Proposed
- Accepted
- Superseded
- Deprecated

## ADR Template

- Title
- Date
- Status
- Context
- Decision
- Consequences
- Alternatives considered
- Rollout plan
- Rollback plan
